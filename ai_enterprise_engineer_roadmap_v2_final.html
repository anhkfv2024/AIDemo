🧱 Module 1: Python + LLM Thinking

Tuần 1 - 3
📅 Tuần 1: Nền tảng lập trình cho AI
  • Setup Môi trường & Python cơ bảnCài đặt Python, Virtual Environment (venv). Ôn tập cú pháp cơ bản.
  • Thư viện Requests & JSONCách gọi REST API thủ công, parse JSON response.
  • Quản lý biến môi trường (.env)Sử dụng python-dotenv để bảo mật API keys. Tuyệt đối không hardcode.
  • Lập trình Bất đồng bộ (Async Python)Hiểu cơ bản về async/await để xử lý I/O bound (gọi API không bị block).
  • Tổng hợp Tuần 1Viết 1 script nhỏ gọi API lấy thời tiết (hoặc public API) dùng async và lưu key trong .env.
📅 Tuần 2: Gọi LLM API & LiteLLM
  • Làm quen LiteLLMCài đặt và setup LiteLLM. Hiểu tại sao cần unify API.
  • Multi-LLM CallerViết script gọi luân phiên OpenAI, Gemini, Anthropic bằng cùng 1 cấu trúc code.
  • Token & Cost TrackingSử dụng hàm của LiteLLM để lấy số token đầu vào/đầu ra và tính chi phí ($).
  • Khám phá ParametersThử nghiệm thay đổi Temperature, Max Tokens, Top-p. Ghi nhận sự khác biệt.
  • Xử lý Error & TimeoutBắt lỗi Rate Limit, Timeout khi gọi API LLM.
📅 Tuần 3: Foundation - Toán & Xác suất của LLM
  • Xác suất cơ bản (Probability)Ôn lại xác suất rời rạc. Hiểu rằng LLM dự đoán từ bằng xác suất.
  • Token ProbabilityCách LLM tính điểm (logits) cho từng token tiếp theo trong từ điển.
  • Softmax IntuitionHàm Softmax biến đổi điểm logits thành phần trăm xác suất (0-1).
    softmax(z_i) = exp(z_i) / Σ exp(z_j)
  • Temperature MathHiểu toán học của Temperature: T > 1 làm mượt phân phối, T < 1 làm nhọn phân phối.
  • Đóng gói Module 1Review lại toàn bộ code & khái niệm.

🎯 OUTPUT CẦN ĐẠT MODULE 1

  • Script Multi-LLM caller hoạt động ổn định.
  • Hàm Token counter tự động.
  • Cost tracking logger (ghi file log chi phí mỗi lần gọi).

🧱 Module 2: Prompting + Structured Output

Tuần 4 - 5
📅 Tuần 4: Prompt Engineering & Data Extraction
  • Prompt Engineering cơ bảnZero-shot, Few-shot prompting, Role-playing (System Prompt).
  • JSON Schema DesignThiết kế cấu trúc JSON mong muốn để trích xuất dữ liệu.
  • Structured Output APISử dụng response_format={ "type": "json_object" } hoặc Pydantic với LiteLLM.
  • Extraction Task (Email Thương mại)Viết prompt xử lý Email kinh doanh trích xuất: Người gửi, Mục đích, Deadline.
  • Chain of Thought (CoT)Thêm bước "hãy suy nghĩ từng bước" trước khi trả về JSON để tăng độ chính xác.
📅 Tuần 5: Giảm Ảo Giác & Foundation Models
  • Hallucination Reduction TechniquesKỹ thuật "Nếu không biết, hãy trả lời là Không tìm thấy thông tin".
  • Context Window Deep DiveHiểu giới hạn cửa sổ ngữ cảnh. Chuyện gì xảy ra khi vượt quá giới hạn hoặc nhồi nhét quá nhiều?
  • Tokenization InternalsThử nghiệm với Tiktoken. Phân tích chi phí token đối với các cấu trúc ngôn ngữ khác nhau.
  • Transformer IntuitionĐọc/xem giải thích trực quan về cơ chế Attention (không cần code lại từ đầu, chỉ cần hiểu).
  • Hoàn thiện PipelineĐóng gói script nhận input là Text, trả về JSON chuẩn xác 100%.

🎯 OUTPUT CẦN ĐẠT MODULE 2

  • Công cụ Email → JSON extractor chạy ổn định.
  • Stable prompt template (có thể tái sử dụng cho các form data khác).

🧱 Module 3: Embedding + Vector Search

Tuần 6 - 8
📅 Tuần 6: Embedding & Vector Math
  • Khái niệm EmbeddingBiến text thành dãy số (vector). Dùng API lấy embedding (vd: text-embedding-3-small).
  • Toán Vector (Linear Algebra)Hiểu vector là gì trong không gian N chiều. Phép cộng, trừ vector.
  • Toán Matrix cơ bảnHiểu ma trận là tập hợp các vector. Phép nhân vô hướng (Dot Product).
  • Cosine Similarity MathHiểu công thức:
    cos(θ) = (A · B) / (||A|| * ||B||)
  • Code Cosine Similarity NumpyViết hàm tính khoảng cách giữa 2 câu bằng numpy thuần túy để hiểu bản chất.
📅 Tuần 7: Chunking & pgvector Setup
  • Chunking StrategiesCách cắt nhỏ tài liệu: theo ký tự, theo từ, theo đoạn văn (Recursive Character Splitter).
  • Setup PostgreSQLCài đặt PostgreSQL (hoặc dùng Docker).
  • Cài đặt Extension pgvectorEnable extension vector trong PostgreSQL. Tạo bảng lưu trữ embedding.
  • Insert Data vào pgvectorViết script đọc text, gọi API tạo embedding, insert vào database.
  • Thực thi Semantic Search (SQL)Dùng toán tử <=> (cosine distance) trong pgvector để tìm kiếm.
📅 Tuần 8: Xử lý Document Doanh Nghiệp
  • Đọc file PDFSử dụng PyMuPDF (fitz) để đọc text từ tài liệu nội bộ.
  • Xử lý Ngôn ngữ đặc thùPhân tích văn bản tiếng Việt có dấu, định dạng bảng biểu. Test các chunk size phù hợp.
  • Build Search FlowUser input câu hỏi -> Embed câu hỏi -> Query pgvector -> Trả về top 3 chunks.
  • Tối ưu Index DBLàm quen với HNSW index trong pgvector để tăng tốc độ search.
  • Kiểm thử độ chính xácĐánh giá xem search bằng vector có tìm ra đúng đoạn văn cần thiết không.

🎯 OUTPUT CẦN ĐẠT MODULE 3

  • API / Script Semantic Search cho file PDF.
  • Retrieval chuẩn xác cho tài liệu doanh nghiệp.

🧱 Module 4: RAG API Backend

Tuần 9 - 11
📅 Tuần 9: Xây dựng Backend với FastAPI
  • FastAPI FoundationKhởi tạo project FastAPI, Pydantic models cho request/response.
  • Upload File EndpointTạo API endpoint cho phép client upload file PDF.

📅 Tuần 10: RAG Flow (Retrieve & Generate)

  • Prompt for RAGThiết kế system prompt: "Chỉ trả lời dựa trên context được cung cấp..."
  • Citation (Trích dẫn nguồn)Yêu cầu LLM format câu trả lời kèm theo ID hoặc page number của chunk được dùng.
  • Xử lý Lịch sử Chat (Memory)Lưu trữ mảng messages để model có ngữ cảnh của các câu hỏi trước.
  • Refactor & Tổ chức CodeTách routers, services, database config (Chuẩn bị tư duy System).
📅 Tuần 11: Foundation - Ranking & BM25
  • Information Retrieval BasicsSự khác biệt giữa Keyword search và Semantic search.
  • BM25 IntuitionHiểu thuật toán TF-IDF và BM25 (chấm điểm từ khóa hiếm/phổ biến).
  • Giới thiệu Hybrid SearchConcept kết hợp BM25 + Vector Search để cho kết quả hoàn hảo nhất.
  • Hoàn thiện API RAGXử lý lỗi, thêm logging.
  • Đóng gói Docker (Optional)Viết Dockerfile để containerize FastAPI app.

🎯 OUTPUT CẦN ĐẠT MODULE 4

  • Backend API RAG hoàn chỉnh (Sẵn sàng cho frontend kết nối).
  • Pipeline có trích dẫn nguồn văn bản (citation).

🧱 Module 5: ML Foundation for AI Engineer

Tuần 12 - 15
📅 Tuần 12: Machine Learning Cơ Bản
  • Giới thiệu ML FlowSự khác biệt của ML truyền thống so với gọi API LLM.
  • Regression (Hồi quy)Dự đoán một giá trị liên tục. Code mẫu Linear Regression (Scikit-learn).
  • Classification (Phân loại)Dự đoán nhãn rời rạc (vd: Spam/Not Spam). Logistic Regression.
  • Data PreprocessingXử lý null, chuẩn hóa dữ liệu (StandardScaler).
  • Thực hành Scikit-learnTrain model cơ bản trên tập dataset tabular có sẵn.
📅 Tuần 13: Feature & Data Splitting
  • Feature EngineeringCách chọn lọc và biến đổi các đặc trưng để model học tốt hơn.
  • Train / Validation / Test SplitNguyên tắc không bao giờ test model trên tập dữ liệu đã train.
  • Metrics Đánh Giá Phân LoạiAccuracy, Precision, Recall, F1-Score. Hiểu trade-off.
  • Cross-ValidationK-fold cross validation để đánh giá model khách quan.
  • Project: Text Classification DataChuẩn bị dataset (ví dụ: Spam SMS) để huấn luyện.
📅 Tuần 14: Toán Học của Việc "Học"
  • Loss Function (Hàm suy hao)Công cụ đo lường sai số của model. MSE, Cross-Entropy.
  • Đạo hàm cơ bản (Calculus)Ôn tập đạo hàm: tốc độ thay đổi và chiều hướng.
  • Gradient Descent (Cốt lõi)Hiểu cách model tự sửa sai.
    θ := θ - α∇J(θ)
  • Learning Rate (α)Bước nhảy. Nhảy quá to (overshoot), quá nhỏ (lâu tới đích).
  • Trực quan hóa quá trình hội tụVẽ đồ thị Loss function giảm dần theo các epoch.
📅 Tuần 15: Overfitting & Đóng gói Model
  • Overfitting vs UnderfittingHiện tượng "học vẹt" của model và cách nhận biết.
  • Regularization techniquesLasso (L1), Ridge (L2) cơ bản để chống học vẹt.
  • Huấn luyện Spam ClassifierTrain model TF-IDF + Naive Bayes/Random Forest phân loại văn bản.
  • Lưu model (Pickle/Joblib)Cách xuất model ra file để dùng lại mà không cần train lại.
  • Tích hợp vào FastAPITạo endpoint `/predict` load model đã train và trả về kết quả Classification.

🎯 OUTPUT CẦN ĐẠT MODULE 5

  • Mini-project: Tabular Prediction chạy thực tế.
  • Nắm vững bản chất Loss Function & Gradient Descent.

🧱 Module 6: AI System Thinking

Tuần 16 - 18
📅 Tuần 16: Tối ưu Độ trễ (Latency) & Caching
  • Phân tích Latency của LLMTại sao TTFT (Time To First Token) lại quan trọng cho UX?
  • Streaming ResponseSửa endpoint FastAPI RAG để trả về dạng Stream (từng chữ một).
  • Semantic Caching - ConceptKhông gọi LLM nếu user hỏi câu có ý nghĩa tương đương câu đã hỏi.
  • Triển khai Caching (Redis/In-memory)Code logic check khoảng cách vector câu hỏi, nếu < ngưỡng -> Cache Hit.
  • Rate LimitingBảo vệ API của bạn khỏi abuse. Chặn giới hạn request/phút.
📅 Tuần 17: Tính Ổn định & Bất đồng bộ
  • Thiết kế Retry MechanismSử dụng thư viện tenacity để tự động retry khi API LLM bị lỗi.
  • FallbacksNếu OpenAI lỗi, tự động chuyển sang gọi Gemini/Claude qua cấu hình LiteLLM.
  • Message Queues ConceptRabbitMQ hoặc Celery: Xử lý file PDF nặng mà không crash server.
  • Log System (Observability)Ghi log request, response, cost, latency chuẩn chỉ vào file.
  • Security & API KeysCách quản lý Key tập trung, luân chuyển key (Key rotation).
📅 Tuần 18: Tích hợp & Ôn Tập Tổng Thể
  • Code Review Toàn Hệ ThốngDọn dẹp mã nguồn, check lại các module ghép nối.
  • Tối ưu Deployment WorkflowSetup CI/CD cơ bản hoặc cấu trúc thư mục chuẩn cho VPS deployment.
  • Test luồng End-to-EndTừ upload PDF -> Parsing Queue -> Insert DB -> Chat Stream -> LLM Fallback.
  • Documentation (Tài liệu hóa)Viết README.md và Swagger UI cho API một cách chuyên nghiệp.
  • Định hướng Bước Tiếp TheoLựa chọn specialization (Vision, LLM Internals, hoặc Agent Workflows).

  • Production-style AI API (sẵn sàng chịu tải cho nội bộ doanh nghiệp).
